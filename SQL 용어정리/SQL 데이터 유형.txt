VARCHAR 와 VARCHAR2 차이

VARCHAR : Oracle 9i 이전 버전에서 사용함
          저장되는 문자열의 실제 길이에 따라 크기가 결정된다
          문자열 길이에 상한 제한이 없다
          따라서 저장공간을 효율적으로 사용하지 못할 수도 있다
      예를 들어, 데이터 길이가 10자인 VARCHAR열에 "Hello"를 저장하면 "Hello"가 저장되고 나머지 공간은 비어있게 된다.

VARCHAR2 : Oracle 9i 이후 버전부터 사용되는 문자열 데이터 유형
           실제 데이터 길이와 관계없이 미리 정의된 최대 크기로 저장 공간을 사용한다.
           문자열 길이에 제한을 둬서 저장공간을 효율적으로 사용할 수 있다.
      예를 들어, 데이터 길이가 10자인 VARCHAR2 열에 "Hello"를 저장하면 "Hello"가 저장된다.

(둘 다 null 값을 허용한다)



INT 와 NUMBER 차이
 소수점값을 저장할 수 있는지 없는지

INT : 정수 유형 데이터
      소수점 이하 값을 저장할 수 없으며, 정수 값만 저장할 수 있다.
    예를 들어, 1,2,-3과 같은 정수값을 저장할 때 사용된다.

NUMBER : 소수점 이하 값을 저장할 수 있는 숫자 데이터 유형
         정수와 소수 모두를 저장할 수 있다.
    예를 들어, NUMBER(10,2)는 총 10자리 숫자를 표현하고 소수점 이하 2자리까지 값을 저장할 수 있게되는 것이다.



LOB(Large Object : 대용량 객체) - 데이터 유형
    텍스트나 이진 데이터와 같은 대량의 데이터를 저장하는데 사용된다.
    종류는 CLOB, BLOB이 있다

  CLOB(Character Large Object) 
    문자 데이터를 저장하는데 사용한다
    대용량 텍스트 문서를 저장하고 검색하는데 적합
    소설, 웹 페이지, 텍스트, 뉴스기사 등의 텍스트를 저장
  BLOB(Binary Large Object)
    바이너리 데이터(=이진 데이터)를 저장하는데 사용된다.
    이미지, 비디오, 오디오, 압축된 파일 등과 같은 이진 형식의 데이터를 저장하는데 적합하다.
